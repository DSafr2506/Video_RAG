# Производительность и оптимизация

## Бенчмарки

### Тестовое окружение
- GPU: NVIDIA T4 (16GB)
- CPU: Intel Xeon (8 cores)
- RAM: 32GB
- Dataset: 436 видео, ~13000 чанков

### Результаты

| Этап | CPU | GPU | Ускорение |
|------|-----|-----|-----------|
| Создание эмбеддингов | 15 мин | 2.5 мин | 6x |
| Построение индексов | 30 сек | 30 сек | 1x |
| Обработка запросов | 2 мин | 20 сек | 6x |
| **Всего** | **17.5 мин** | **3.3 мин** | **5.3x** |

## Оптимизации

### 1. Батчевая обработка
```python
# Плохо
for text in texts:
    embedding = model.encode(text)

# Хорошо
embeddings = model.encode(texts, batch_size=32)
```
Ускорение: 10-15x

### 2. HNSW индексы
```python
# Вместо Flat индекса
index = faiss.IndexHNSWFlat(dim, 32)
```
Ускорение поиска: 100-1000x на больших данных

### 3. Группировка по топикам
Уменьшает пространство поиска в 10-20 раз

### 4. GPU ускорение
```python
model.to("cuda")
```
Ускорение: 5-10x

## Рекомендации

### Для маленьких датасетов (<1000 видео)
- CPU достаточно
- batch_size = 16
- Простой Flat индекс

### Для средних датасетов (1000-10000 видео)
- Используйте GPU
- batch_size = 32
- HNSW индекс

### Для больших датасетов (>10000 видео)
- Обязательно GPU
- batch_size = 64
- HNSW индекс с большим M
- Рассмотрите распределенную обработку

## Узкие места

1. **Создание эмбеддингов** - самая медленная часть
   - Решение: GPU, больший batch_size
   
2. **Загрузка модели** - занимает время при старте
   - Решение: кэширование модели
   
3. **Память GPU** - может закончиться на больших батчах
   - Решение: уменьшить batch_size
