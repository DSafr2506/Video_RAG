# FAQ - Часто задаваемые вопросы

## Общие вопросы

### Что такое Video RAG?
RAG (Retrieval-Augmented Generation) - это подход, при котором система сначала находит релевантную информацию, а затем использует её. В нашем случае мы ищем фрагменты видео по текстовым запросам.

### Какие языки поддерживаются?
Модель BGE-M3 поддерживает множество языков, включая русский, английский, китайский и другие.

### Нужна ли GPU?
Нет, система работает и на CPU, но GPU значительно ускоряет обработку (в 5-10 раз).

## Технические вопросы

### Как изменить модель эмбеддингов?
Отредактируйте `EMBED_MODEL` в `config.py`:
```python
EMBED_MODEL = "sentence-transformers/all-MiniLM-L6-v2"
```

### Как увеличить точность?
Попробуйте:
- Увеличить `RETRIEVE_TOPK` (больше кандидатов)
- Уменьшить `MAX_TOKENS` (более мелкие чанки)
- Увеличить `OVERLAP` (больше контекста)

### Почему результаты не точные?
Возможные причины:
- Плохое качество транскриптов
- Неподходящая модель для вашего языка
- Слишком общие запросы
- Недостаточно данных для обучения

### Как ускорить обработку?
- Используйте GPU
- Увеличьте batch_size
- Уменьшите количество видео
- Используйте более быструю модель

## Проблемы и решения

### Out of Memory
Уменьшите batch_size в `embeddings.py`:
```python
embeddings = model.embed_texts(texts, batch_size=16)
```

### Медленная работа
- Проверьте, используется ли GPU
- Уменьшите `RETRIEVE_TOPK`
- Используйте более легкую модель

### Ошибки импорта
Убедитесь, что все зависимости установлены:
```bash
pip install -r requirements.txt
```

## Разработка

### Как добавить новую функцию?
1. Создайте новый модуль в корне проекта
2. Импортируйте в `main.py`
3. Добавьте тесты в `tests/`
4. Обновите документацию

### Как протестировать изменения?
```bash
make test
```

### Где хранятся данные?
В директории `data/`. Добавьте её в `.gitignore`, чтобы не коммитить большие файлы.
