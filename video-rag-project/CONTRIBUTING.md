# Contributing Guidelines

Спасибо за интерес к проекту! Мы рады любому вкладу.

## Как внести вклад

### Сообщения об ошибках

Если вы нашли баг:
1. Проверьте, не создан ли уже issue
2. Создайте новый issue с подробным описанием
3. Укажите шаги для воспроизведения
4. Приложите логи и скриншоты

### Предложения

Есть идея для улучшения?
1. Создайте issue с тегом "enhancement"
2. Опишите предлагаемую функциональность
3. Объясните, зачем это нужно

### Pull Requests

1. Форкните репозиторий
2. Создайте ветку для вашей фичи (`git checkout -b feature/amazing-feature`)
3. Внесите изменения
4. Добавьте тесты, если необходимо
5. Убедитесь, что код соответствует стилю проекта
6. Закоммитьте изменения (`git commit -m 'Add amazing feature'`)
7. Запушьте в ветку (`git push origin feature/amazing-feature`)
8. Откройте Pull Request

## Стиль кода

- Следуйте PEP 8
- Используйте понятные имена переменных
- Добавляйте комментарии для сложной логики
- Пишите docstrings для функций и классов

## Тестирование

Перед отправкой PR убедитесь, что:
- Все существующие тесты проходят
- Добавлены тесты для новой функциональности
- Код работает на Python 3.8+

## Коммиты

Используйте понятные сообщения коммитов:
- `feat: добавлена новая функция`
- `fix: исправлена ошибка в ...`
- `docs: обновлена документация`
- `refactor: рефакторинг модуля ...`
- `test: добавлены тесты для ...`

## Вопросы?

Не стесняйтесь задавать вопросы в issues!
