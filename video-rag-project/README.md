# Video RAG System

Система поиска релевантных фрагментов видео на основе текстовых запросов с использованием векторного поиска и семантических эмбеддингов.

## Описание

Проект реализует baseline решение для задачи поиска фрагментов в видео по текстовым запросам. Система использует транскрипты видео, разбивает их на чанки, создает векторные представления и выполняет семантический поиск с помощью FAISS.

## Архитектура

- **Chunking**: Разбиение транскриптов на перекрывающиеся фрагменты
- **Embeddings**: Создание векторных представлений с помощью BGE-M3
- **Indexing**: HNSW индексы для быстрого поиска
- **Retrieval**: Двухэтапный поиск с расширением временных окон

## Использование

1. Подготовьте данные в директории `data/`:
   - `transcripts.pkl` - транскрипты видео
   - `video_topic.csv` - соответствие видео и топиков
   - `test_topic.csv` - тестовые запросы

2. Запустите основной скрипт:

```bash
python main.py
```

3. Результат сохранится в `submission.csv`

## Параметры

Основные параметры настраиваются в `config.py`:

- `MAX_TOKENS`: Максимальное количество токенов в чанке (80)
- `OVERLAP`: Перекрытие между чанками (30)
- `RETRIEVE_TOPK`: Количество кандидатов для первого этапа (200)
- `FINAL_TOPK`: Финальное количество результатов (5)
- `TEMPORAL_WINDOW`: Расширение временного окна в секундах (6)

## Модели

- **Embedding**: BAAI/bge-m3 - многоязычная модель для создания эмбеддингов
- **Reranking**: BAAI/bge-reranker-large - модель для переранжирования (опционально)

## Особенности

- Поддержка GPU для ускорения вычислений
- Батчевая обработка запросов
- Группировка по топикам для улучшения точности
- Расширение временных окон для учета контекста

## Производительность

Система обрабатывает ~400 видео и ~13000 чанков за несколько минут на GPU.
