# Система поиска фрагментов в видео

Проект для семантического поиска релевантных фрагментов в видео на основе текстовых запросов.

## О проекте

Это решение использует современные подходы к обработке естественного языка для поиска нужных моментов в видео. Система анализирует транскрипты, создает векторные представления и находит наиболее релевантные фрагменты.

## Возможности

- 🎯 Точный семантический поиск
- ⚡ Быстрая обработка благодаря FAISS
- 🌍 Поддержка нескольких языков
- 🔧 Гибкая настройка параметров
- 📊 Группировка по тематикам

## Быстрый старт

### Установка

```bash
# Клонируйте репозиторий
git clone https://github.com/yourusername/video-rag-project.git
cd video-rag-project

# Запустите скрипт установки
./setup.sh
```

### Подготовка данных

Поместите файлы в директорию `data/`:
- `transcripts.pkl` - транскрипты видео
- `video_topic.csv` - соответствие видео и тематик
- `test_topic.csv` - тестовые запросы

### Запуск

```bash
python main.py
```

Результаты сохранятся в `submission.csv`

## Как это работает

1. **Подготовка**: Транскрипты разбиваются на небольшие фрагменты с перекрытием
2. **Векторизация**: Каждый фрагмент преобразуется в вектор с помощью BGE-M3
3. **Индексация**: Векторы добавляются в FAISS индекс для быстрого поиска
4. **Поиск**: Запрос преобразуется в вектор и ищутся похожие фрагменты
5. **Ранжирование**: Результаты сортируются по релевантности

## Настройка

Основные параметры в `config.py`:

```python
MAX_TOKENS = 80          # Размер фрагмента
OVERLAP = 30             # Перекрытие фрагментов
RETRIEVE_TOPK = 200      # Кандидатов на первом этапе
FINAL_TOPK = 5           # Финальных результатов
TEMPORAL_WINDOW = 6      # Расширение временного окна
```

## Требования

- Python 3.8+
- CUDA (опционально, для GPU)
- 8GB RAM минимум

- Создание индексов: ~2 минуты
- Обработка запросов: ~30 секунд
