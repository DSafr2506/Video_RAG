# Архитектура системы

## Обзор

Система Video RAG построена на модульной архитектуре, где каждый компонент отвечает за свою задачу.

## Компоненты

### 1. Data Loader (`data_loader.py`)
Отвечает за загрузку данных из различных источников:
- Транскрипты видео (pickle)
- Соответствие видео-топики (CSV)
- Тестовые запросы (CSV)

### 2. Chunking (`chunking.py`)
Разбивает транскрипты на управляемые фрагменты:
- Контролирует размер чанков по токенам
- Создает перекрытия для сохранения контекста
- Расширяет временные окна

### 3. Embeddings (`embeddings.py`)
Создает векторные представления текста:
- Использует предобученную модель BGE-M3
- Батчевая обработка для эффективности
- Нормализация векторов

### 4. Indexing (`indexing.py`)
Управляет векторными индексами:
- HNSW алгоритм для быстрого поиска
- Настраиваемые параметры точности
- Поддержка больших объемов данных

### 5. Retriever (`retriever.py`)
Выполняет поиск релевантных фрагментов:
- Двухэтапный поиск
- Агрегация по видео
- Ранжирование результатов

### 6. Main Pipeline (`main.py`)
Объединяет все компоненты:
- Оркестрация процесса
- Обработка ошибок
- Генерация результатов

## Поток данных

```
Транскрипты → Chunking → Embeddings → Indexing
                                          ↓
Submission ← Retriever ← Query Embeddings
```

## Оптимизации

1. **Батчевая обработка**: Обработка нескольких запросов одновременно
2. **GPU ускорение**: Использование CUDA для вычислений
3. **HNSW индексы**: Быстрый приближенный поиск
4. **Группировка по топикам**: Уменьшение пространства поиска

## Расширяемость

Система легко расширяется:
- Добавление новых моделей эмбеддингов
- Интеграция reranking моделей
- Кастомные стратегии чанкинга
- Альтернативные индексы
